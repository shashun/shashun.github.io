Composer的常用命令

2019年4月27日
| PHP

init（初始化）

该命令用于创建 composer.json 文件，并进行基础信息配置 可以配置Package name、Description、Author、Minimum、Package Type、License、dependencies 及 dev dependencies 信息。

$ composer init

配置完成以后的信息如下所示：

{
    "name": "demo/demo",
    "description": "demo",
    "authors": [
        {
            "name": "demo",
            "email": "demo@demo.com"
        }
    ],
    "require": {}
}

search（搜索）

根据名称搜索相关的包，成功后会列出符合的相关包的信息， 以monolog/monolog为例

$ composer search monolog/monolog
monolog/monolog Sends your logs to files, sockets, inboxes, databases and various web services
oasis/logging Wrapper of monolog/monolog. Should be easier to use/configure than monolog
bluebik/logger Log seperate folder by level build on monolog/monolog

show（详情）

  1. 无参数时, 输出依赖的模块的列表

    $ composer show
monolog/monolog 1.24.0 Sends your logs to files, sockets, inboxes, databases and various web...
psr/log         1.1.0  Common interface for logging libraries

  2. 有参数时 ，根据包的名称，列出包的相关信息，以 monolog/monolog 为例：

    $ composer show  monolog/monolog
name     : monolog/monolog
descrip. : Sends your logs to files, sockets, inboxes, databases and various web services
keywords : log, logging, psr-3
versions : * 1.24.0
type     : library
license  : MIT License (MIT) (OSI approved) https://spdx.org/licenses/MIT.html#licenseText
source   : [git] https://github.com/Seldaek/monolog.git bfc9ebb28f97e7a24c45bdc3f0ff482e47bb0266
dist     : [zip] https://api.github.com/repos/Seldaek/monolog/zipball/bfc9ebb28f97e7a24c45bdc3f0ff482e47bb0266 bfc9ebb28f97e7a24c45bdc3f0ff482e47bb0266

install (安装)

我们先在 composer.json 配置中添加一个 monolog/monolog 依赖库，如下：

{
    "name": "demo/demo",
    "description": "demo",
    "authors": [
        {
            "name": "demo",
            "email": "demo@demo.com"
        }
    ],
   "require": {
        "monolog/monolog": "^1.24"
    }
}

然后执行如下命令进行安装：

$ composer install

update （更新）

当我们修改了依赖关系，不管是新增，还是修改了版本，又或者是删除，执行此命令

$ composer update  # 从 composer.json 安装最新扩展包和依赖；
$ composer update  monolog/monolog    #指定包更新

require （申明依赖）

增加新的依赖包到当前目录的 composer.json 文件中 ，在添加或改变依赖时， 修改后的依赖关系将被安装或者更新。

$ composer require monolog/monolog
